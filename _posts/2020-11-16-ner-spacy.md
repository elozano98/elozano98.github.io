## Abstract
In this post a custom Named Entity Recognition model is going to be created by using **[spaCy](https://spacy.io/)**: a free, open-source library for advanced Natural Language Processing (NLP) in Python.

![](/assets/img/posts/ner-spacy/spacy.jpg)


<br/>

## What is Named Entity Recognition?
**Named Entity Recognition (NER)** is a subtask of information extraction that **locates** and **classifies** named entities contained in a text. Entities can be classified into a set of pre-defined categories as for example: locations, organizations, products, etc.

![](/assets/img/posts/ner-spacy/ner-example.png)

<br/>

## Data formatting 
When preparing the data to train the model it is necessary to use the spaCy’s format. This formatting uses the starting and ending entity boundaries to locate each entity into their corresponding sentence.

```python
train_data = [
    ("Uber blew through $1 million a week", [(0, 4, 'ORG')]),
    ("Android Pay expands to Canada", [(0, 11, 'PRODUCT'), (23, 30, 'GPE')]),
    ("Spotify steps up Asia expansion", [(0, 8, "ORG"), (17, 21, "LOC")]),
    ("Google Maps launches location sharing", [(0, 11, "PRODUCT")]),
    ("Google rebrands its business apps", [(0, 6, "ORG")])]
```

<br/>

## Updating the Named Entity Recognizer
SpaCy provides an easy way of updating their NER model with your own examples. This can be done from one of the **[existing pre-trained models](https://spacy.io/usage/models)** or from an empty model just by defining the language.

```python
import spacy

model = "en_core_web_sm"

if model is None:
    nlp = spacy.blank("en")  # create blank Language class
    print("Created blank 'en' model")
else:
    nlp = spacy.load(model)  # load existing spaCy model
    print("Loaded model '%s'" % model)
```

After declaring the model, the next step is to get the named entity recognizer from the pipeline. If the model has been created from scratch, this named entity recognizer will not be in the pipeline, so it would be necessary to create it and add it to the model pipeline.

```python
if "ner" in nlp.pipe_names:
    ner = nlp.get_pipe("ner")
else:
    ner = nlp.create_pipe("ner")
    nlp.add_pipe(ner, last=True)
```

In order to update this Named Entity Recognizer with a set of custom entities, the next step is to add the labels of entities contained in the training data.

```python
entities = ['ORG', 'PRODUCT', 'GPE', 'LOC']

for entity_label in entities:
    ner.add_label(entity_label)
```

If the model has not been created from scratch, it is needed to get all pipe names that are not wanted to be trained.

Since only the NER is wanted to be trained, all other pipes will be disabled.

```python
pipe_exceptions = ["ner", "trf_wordpiecer", "trf_tok2vec"]
other_pipes = [pipe for pipe in nlp.pipe_names if pipe not in pipe_exceptions]
```

Before starting training, if a new model is going to be trained, weights should be reset and initialized randomly. Then, this training process is repeated a certain number of times where, in each iteration, all data samples are shuffled and split into batches that update the NER model.

```python
with nlp.disable_pipes(*other_pipes), warnings.catch_warnings():

        warnings.filterwarnings("once", category=UserWarning, module='spacy')

                    annotations,  # batch of annotations

                    losses=losses,
                )
            print("Losses", losses)
```

The **dropout** parameter can be used to difficult the model to memorize the data. So, maybe it is an interesting option to avoid **overfitting**.

Once the NER model has been trained with the custom entities, it should be able to successfully recognizing them. So, lets check it out with a simple example containing some of the custom entities:

```python
example_sentence = "GPT-3 has been developed by OpenAI."

doc = nlp(example_sentence)
print("Entities:", [(ent.text, ent.label_) for ent in doc.ents])

# Entities: [(GPT-3, 'PRODUCT'), (OpenAI, 'ORG')]
```
